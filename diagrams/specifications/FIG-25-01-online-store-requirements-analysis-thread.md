# FIG-25-01: Online Store Requirements Analysis Thread

## Authorisation and status

- Authorisation to proceed without a separate diagram approval pause: author instruction dated 2026-07-11.
- Maximum Codex status: Review.

## Purpose

Show a beginner-readable trace from stakeholder need through requirements and analysis models to design response, assessment and evidence, with feedback that makes refinement iterative. The author instruction dated 2026-07-11 authorises this specification revision and subsequent source generation without a separate approval pause.

## Audience

Beginner architects, analysts, product owners, engineers and testers.

## Question answered

How do analysis artefacts connect a need to credible evidence without implying a one-pass waterfall?

## Notation

Original PlantUML traceability teaching view.

## Required elements

Stable identifiers for one need, one functional requirement, one quality requirement, a use case, conceptual model, design response, verification case and evidence.

## Required relationships

Label every relationship. Show forward trace links and feedback from evidence to requirement refinement.

## Main flow or structure

Need to requirements, analysis models, design response, verification case and evidence.

## Alternative and exception flows

Feedback arrows return evidence to functional and quality requirements for iterative refinement.

## Scope

One Online Store checkout traceability thread at analysis level.

## Exclusions

Detailed implementation, complete requirement catalogues, formal SysML syntax and exhaustive test cases.

## Accessibility requirements

Use a portrait, top-to-bottom layout designed for approximately 700 to 750 pixels of page width, short labels, sufficient contrast and relationship labels. Colour supports grouping but is not the sole carrier of meaning. Avoid overlaps, clipping, small type and excessive crossings.
