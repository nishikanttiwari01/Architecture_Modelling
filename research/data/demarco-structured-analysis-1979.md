## Relevant summary

Structured analysis popularised data flow diagrams as a way to focus on data movement and processing rather than program structure. Chapter 8 uses this source to explain DFDs as practical architecture views, not as a current formal standard for all data modelling work.

## Terminology and version notes

The chapter uses common DFD terms such as external entity, process, data store and data flow. It avoids implying that DFD notation is governed like UML, BPMN or ArchiMate.
