## Relevant findings

The specification defines a language-agnostic interface description for HTTP APIs. An OpenAPI document can describe paths, operations, parameters, request bodies, responses, security requirements and reusable schemas. Chapter 27 uses this only to support machine-readable HTTP contract guidance. It does not claim that an OpenAPI document alone captures all runtime behaviour or governance.
