## Relevant summary

Official tool sources support a practical distinction between semantic BPMN modellers and general drawing tools:

- Camunda Modeler documentation describes modelling tools for BPMN, DMN and related Camunda artefacts. The repository has already used Camunda Desktop Modeler validation results for Chapter 6 BPMN XML.
- Bizagi Modeler is presented by Bizagi as a business process modelling tool with BPMN-oriented modelling and process documentation capabilities.
- diagrams.net is a general-purpose diagramming tool with many shape libraries. It is useful for illustrations and informal diagrams, but it does not make a diagram a semantic BPMN model by itself.
- SAP Signavio and Visual Paradigm provide BPMN modelling capabilities in broader process or modelling suites.
- Camunda Web Modeler is useful when browser-based collaboration and Camunda platform workflow are relevant.

## Terminology and version notes

Chapter 6 should avoid implying that all BPMN-looking diagrams are equivalent. A diagram drawn with BPMN shapes in a generic drawing tool may be useful for communication, but it is not the same as an editable BPMN XML model with process semantics.
