## Relevant summary

The official tool pages support a practical "how to create models" section in Chapter 10. The chapter should distinguish workshop tools, drawing tools, diagrams-as-code tools, contract tooling and catalogue tooling. Tool choice should be tied to the modelling purpose rather than presented as a universal recommendation.

For publication diagrams in this repository, PlantUML remains the editable source format requested for Chapter 10. That choice supports repeatable SVG and PNG generation, but it does not make the diagrams semantic DDD or AsyncAPI repository artefacts.
