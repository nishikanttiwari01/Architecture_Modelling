# Simple Online Store Example

This deliberately small example introduces modelling techniques before banking complexity is added.

## Stable actors

- Customer
- Customer Support Agent
- Payment Provider
- Delivery Partner

## Stable software systems

- Online Store
- Payment Provider System
- Delivery Partner System

## Core capabilities and processes

- Browse catalogue
- Place order
- Take payment
- Fulfil order
- Track delivery
- Handle return

## Core information

- Customer
- Product
- Basket
- Order
- Payment
- Shipment

Do not add unnecessary enterprise complexity. Record any new stable names here before using them across chapters.
