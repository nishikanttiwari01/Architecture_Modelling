# Simple Online Store Example

This deliberately small example introduces modelling techniques before banking complexity is added.

## Stable actors

- Customer
- Customer Support Agent
- Payment Provider
- Delivery Partner

## Stable software systems

- Online Store
- Payment Provider System
- Delivery Partner System

## Stable Online Store internal elements

- Web Application
- API Application
- Order Database
- Notification Worker

These are teaching-level internal elements of the Online Store, not separate software
systems. Chapters may introduce more detailed components locally when the distinction is
made explicit.

## Core capabilities and processes

- Browse catalogue
- Place order
- Take payment
- Fulfil order
- Track delivery
- Handle return

## Core information

- Customer
- Product
- Basket
- Order
- Payment
- Shipment

Do not add unnecessary enterprise complexity. Record any new stable names here before using them across chapters.
