# How to Use This Book

## For a complete beginner

Read Parts I and II in order, practise with the simple online store, then use Part III as a diagram-selection guide.

## For a working architect

Use Part III and Part VI as references, then return to specific notation chapters when more depth is needed.

## For a banking architect

Read the foundational chapters first, then work through Part V to see how BIAN combines with business-process, enterprise, software, data and deployment models.

## Chapter conventions

Each major chapter includes:

- The question the model answers
- The intended audience
- Essential concepts and notation
- A simple example
- A banking example
- Comparisons
- Common mistakes
- A review checklist
- A practical exercise

## Model files and exercises

Editable diagram sources and example artefacts are maintained in the repository folders described in `README.md`.
